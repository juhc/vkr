Тема ВКР
Киберучения в рамках курса Защита ОС (подготовка площадки, заданий )

Актуальность
В современном мире число кибератак и уязвимостей постоянно растёт.
Многие инциденты происходят из-за:
эксплуатации давно известных уязвимостей,


человеческого фактора — ошибок администрирования,


некорректной конфигурации систем.


Подготовка специалистов по кибербезопасности требует практических навыков защиты операционных систем.
Создание изолированной учебной площадки (киберполигона) позволяет безопасно отрабатывать реальные сценарии атак и защиты, не затрагивая рабочие сети.

Цель работы
Создать автоматизированную инфраструктуру киберполигона для курса «Защита ОС»,
 которая позволит быстро и единообразно развертывать учебные сценарии киберучений с использованием принципов Infrastructure as Code.

Цель данной работы заключается в разработке реалистичных учебных инфраструктур для курса «Защита ОС», имитирующих работу небольших организаций с типовыми нарушениями и ошибками администрирования, с возможностью автоматизированного развертывания и использования на Киберполигоне ТюмГУ

Задачи
Изучить существующие решения для киберучений и IaC-инструменты.


Определить типовые уязвимости и ошибки администрирования для учебных сценариев.


Спроектировать архитектуру изолированной среды киберполигона.


Реализовать автоматическое развертывание сценариев с помощью IaC.


Создать каталог знаний об уязвимостях и методические материалы для обучения.



Видимые результаты
Изолированный киберполигон, разворачиваемый автоматически из репозитория.


Набор учебных сценариев, моделирующих реальные инциденты и ошибки конфигурации.


Каталог знаний по уязвимостям и типовым ошибкам администрирования.


Методические материалы для преподавателей и студентов.



Практическая значимость
Разработанная платформа позволит:
повысить качество практической подготовки специалистов по защите ОС;


унифицировать учебный процесс;


сократить время на создание и обновление лабораторных стендов;


безопасно отрабатывать реальные сценарии атак и реагирования.

Актуальность темы
В современном мире количество кибератак и уязвимостей неуклонно растёт.
Многие инциденты происходят не из-за сложных хакерских атак, а из-за человеческих ошибок — неправильных настроек, устаревшего программного обеспечения, слабых паролей и отсутствия базовых мер защиты.
Особенно уязвимы небольшие организации, где часто нет выделенных специалистов по безопасности, и администрирование выполняется «по остаточному принципу».
 Такие ошибки приводят к серьёзным последствиям — утечкам данных и потере работоспособности систем.
Для качественной подготовки специалистов по кибербезопасности недостаточно только теории — необходимо отрабатывать практические сценарии, максимально приближенные к реальным условиям.
 Киберполигоны позволяют безопасно моделировать такие ситуации, не затрагивая реальные сети.
 Именно поэтому создание учебных инфраструктур, имитирующих типичные ошибки и уязвимости, является актуальной задачей.

Цель работы
Цель моей работы —
 разработать набор учебных инфраструктурных сценариев, моделирующих инфраструктуру небольших организаций с типичными ошибками администрирования и уязвимостями, с возможностью их автоматического развертывания на Киберполигоне ТюмГУ.

Задачи работы
Для достижения этой цели были решены следующие задачи:
Определены типовые уязвимости и ошибки администрирования, характерные для небольших организаций.


Разработаны учебные инфраструктуры, моделирующие конфигурационные ошибки и угрозы безопасности.


Реализовано автоматизированное развертывание сценариев с использованием принципов Infrastructure as Code.


Подготовлена документация, каталог уязвимостей и методические материалы для обучения.


Проведено тестирование эффективности обучения на основе разработанных сценариев.



Видимые результаты
В результате работы были получены следующие результаты:
Созданы учебные инфраструктуры, моделирующие реальные ошибки администрирования и уязвимости.


Реализовано автоматизированное развертывание на существующем Киберполигоне ТюмГУ.


Подготовлены документация, каталог уязвимостей и методические материалы.


Проведено тестирование, подтвердившее эффективность использования сценариев в учебном процессе.
