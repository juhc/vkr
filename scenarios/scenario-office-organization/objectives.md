# Цели обучения - Сценарий: Офисная организация

## Общие цели

Сценарий "Офисная организация" разработан для моделирования типичной инфраструктуры небольшой компании с реалистичными уязвимостями и ошибками конфигурации. Этот сценарий позволяет студентам отработать навыки обнаружения, эксплуатации и устранения различных типов уязвимостей в комплексной среде.

## Цели обучения по уровням

### Начальный уровень

#### 1. Обнаружение базовых уязвимостей
**Цель**: Научиться находить типичные ошибки конфигурации

**Навыки**:
- Использование сканеров портов (Nmap)
- Обнаружение слабых паролей
- Выявление открытых портов и сервисов
- Анализ конфигурационных файлов

**Практические задачи**:
- Найти все открытые порты на серверах
- Обнаружить слабые пароли через брутфорс
- Выявить небезопасные конфигурации сервисов
- Найти учетные данные в открытом виде

#### 2. Базовые техники эксплуатации
**Цель**: Научиться использовать простые техники атак

**Навыки**:
- Использование учетных данных по умолчанию
- Доступ к открытым файловым шарам
- Подключение к базам данных со слабыми паролями
- Использование открытых веб-интерфейсов

**Практические задачи**:
- Получить доступ к файловому серверу через открытые шары
- Подключиться к базе данных с учетными данными по умолчанию
- Извлечь данные из открытых директорий
- Использовать открытый интерфейс мониторинга

### Средний уровень

#### 3. Эксплуатация веб-уязвимостей
**Цель**: Научиться эксплуатировать уязвимости веб-приложений

**Навыки**:
- Обнаружение и эксплуатация Log4Shell
- Анализ версий библиотек и зависимостей
- Создание и использование эксплойтов
- Получение обратной оболочки

**Практические задачи**:
- Обнаружить уязвимость Log4Shell на веб-сервере
- Создать JNDI payload для эксплуатации
- Получить обратную оболочку через Log4Shell
- Повысить привилегии на скомпрометированном сервере

#### 4. Сетевое перемещение
**Цель**: Научиться перемещаться по сети после первоначальной компрометации

**Навыки**:
- Перечисление сети с компрометированного хоста
- Использование скомпрометированных учетных данных
- Построение карты сети
- Латеральное перемещение

**Практические задачи**:
- Собрать информацию о сети с веб-сервера
- Использовать учетные данные для доступа к другим системам
- Переместиться от веб-сервера к базе данных
- Извлечь данные из базы данных

#### 5. Пост-эксплуатация
**Цель**: Научиться собирать информацию и поддерживать доступ

**Навыки**:
- Сбор информации о системе
- Поиск чувствительных данных
- Создание постоянного доступа
- Сокрытие следов активности

**Практические задачи**:
- Собрать информацию о системе и сети
- Найти учетные данные в файлах
- Создать обратную оболочку для постоянного доступа
- Очистить логи от следов активности

### Продвинутый уровень

#### 6. Комплексные атаки
**Цель**: Научиться комбинировать различные техники для комплексных атак

**Навыки**:
- Построение цепочек атак
- Использование нескольких векторов атаки
- Обход систем защиты
- Координация многоэтапных атак

**Практические задачи**:
- Провести комплексный пентест инфраструктуры
- Использовать цепочку уязвимостей для компрометации критичных систем
- Обойти ограничения безопасности
- Достичь критичных целей (Domain Controller, база данных)

#### 7. Обнаружение и реагирование на инциденты
**Цель**: Научиться обнаруживать и расследовать инциденты безопасности

**Навыки**:
- Анализ логов для обнаружения атак
- Идентификация индикаторов компрометации
- Расследование инцидентов
- Документирование инцидентов

**Практические задачи**:
- Обнаружить признаки атаки в логах
- Идентифицировать скомпрометированные системы
- Провести расследование инцидента
- Составить отчет об инциденте

#### 8. Устранение уязвимостей
**Цель**: Научиться эффективно устранять уязвимости и усиливать безопасность

**Навыки**:
- Применение патчей безопасности
- Исправление конфигураций
- Усиление систем
- Валидация исправлений

**Практические задачи**:
- Устранить все обнаруженные уязвимости
- Настроить безопасные конфигурации
- Применить меры усиления безопасности
- Проверить эффективность исправлений

## Результаты обучения

После завершения всех упражнений студенты должны уметь:

### Технические навыки
1. **Обнаружение уязвимостей**
   - Использовать различные инструменты для сканирования
   - Анализировать результаты сканирования
   - Идентифицировать уязвимости по признакам

2. **Эксплуатация уязвимостей**
   - Использовать готовые эксплойты
   - Адаптировать эксплойты под конкретную среду
   - Создавать простые эксплойты

3. **Сетевое перемещение**
   - Перечислять сети
   - Использовать скомпрометированные учетные данные
   - Перемещаться между системами

4. **Пост-эксплуатация**
   - Собирать информацию
   - Поддерживать доступ
   - Скрывать следы

5. **Устранение уязвимостей**
   - Применять патчи
   - Исправлять конфигурации
   - Валидировать исправления

### Мягкие навыки
1. **Аналитическое мышление**
   - Анализировать сложные ситуации
   - Выявлять взаимосвязи
   - Принимать решения на основе данных

2. **Документирование**
   - Ведение записей о процессе
   - Составление отчетов
   - Коммуникация результатов

3. **Работа в команде**
   - Координация действий
   - Распределение задач
   - Совместное решение проблем

## Критерии оценки

### Отлично (5)
- Обнаружил все уязвимости в инфраструктуре
- Успешно эксплуатировал все уязвимости
- Провел комплексный пентест с полной компрометацией
- Полностью устранил все уязвимости
- Составил подробный отчет с рекомендациями

### Хорошо (4)
- Обнаружил большинство уязвимостей
- Успешно эксплуатировал основные уязвимости
- Провел пентест с частичной компрометацией
- Устранил большинство уязвимостей
- Составил хороший отчет

### Удовлетворительно (3)
- Обнаружил основные уязвимости
- Частично эксплуатировал уязвимости
- Провел базовый пентест
- Устранил некоторые уязвимости
- Составил базовый отчет

### Неудовлетворительно (2)
- Не обнаружил уязвимости или обнаружил минимальное количество
- Не смог эксплуатировать уязвимости
- Не провел пентест
- Не устранил уязвимости
- Не составил отчет или отчет не соответствует требованиям

## Временные рамки

### Начальный уровень
- Обнаружение уязвимостей: 1-2 часа
- Базовые техники эксплуатации: 1-2 часа
- **Итого**: 2-4 часа

### Средний уровень
- Эксплуатация веб-уязвимостей: 2-3 часа
- Сетевое перемещение: 1-2 часа
- Пост-эксплуатация: 1-2 часа
- **Итого**: 4-7 часов

### Продвинутый уровень
- Комплексные атаки: 3-4 часа
- Обнаружение и реагирование: 2-3 часа
- Устранение уязвимостей: 2-3 часа
- **Итого**: 7-10 часов

### Полный сценарий
- **Общее время**: 13-21 час (в зависимости от уровня подготовки)

## Рекомендации по проведению

### Подготовка
1. Убедитесь, что все серверы развернуты и настроены
2. Проверьте доступность всех систем
3. Подготовьте инструкции для студентов
4. Настройте систему мониторинга (опционально)

### Проведение
1. Предоставьте четкие инструкции по началу работы
2. Обеспечьте техническую поддержку
3. Мониторьте прогресс студентов
4. Предоставляйте подсказки при необходимости

### Завершение
1. Проведите разбор результатов
2. Обсудите ключевые моменты
3. Ответьте на вопросы
4. Соберите обратную связь
