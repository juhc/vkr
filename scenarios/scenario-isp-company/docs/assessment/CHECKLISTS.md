# Чек-листы для проверки студентов

## Общие инструкции

Чек-листы предназначены для проверки навыков студентов в обнаружении, эксплуатации и устранении уязвимостей в инфраструктуре ISP компании.

### Критерии оценки

- ✅ **Выполнено полностью** - студент успешно выполнил задачу
- ⚠️ **Выполнено частично** - студент выполнил задачу с ошибками или не полностью
- ❌ **Не выполнено** - студент не смог выполнить задачу

### Балльная система

- **Отлично (5)**: 90-100% задач выполнено полностью
- **Хорошо (4)**: 75-89% задач выполнено полностью
- **Удовлетворительно (3)**: 60-74% задач выполнено полностью
- **Неудовлетворительно (2)**: Менее 60% задач выполнено

## Инструкции для преподавателя

### Проверка работ студентов

1. **Проверка обнаружения**:
   - Попросите студентов показать доказательства (скриншоты, логи)
   - Проверьте, что все уязвимости найдены
   - Оцените качество разведки

2. **Проверка эксплуатации**:
   - Попросите студентов продемонстрировать эксплуатацию
   - Проверьте, что доступ действительно получен
   - Оцените технические навыки

3. **Проверка устранения**:
   - Проверьте, что уязвимости действительно устранены
   - Попросите студентов продемонстрировать валидацию
   - Оцените понимание безопасности

4. **Проверка отчета**:
   - Проверьте полноту и структурированность
   - Оцените техническую грамотность
   - Проверьте рекомендации

### Типичные ошибки студентов

1. **Неполная разведка**: Студенты пропускают важные шаги
2. **Неправильная эксплуатация**: Неправильное использование инструментов
3. **Неполное устранение**: Устраняют только часть уязвимостей
4. **Плохая документация**: Неполные или неструктурированные отчеты

### Рекомендации по оценке

- **Будьте справедливы**: Учитывайте уровень подготовки студента
- **Поощряйте инициативу**: Дополнительные баллы за нестандартные решения
- **Обратная связь**: Давайте конструктивную обратную связь
- **Практичность**: Оценивайте практические навыки, а не только теорию
