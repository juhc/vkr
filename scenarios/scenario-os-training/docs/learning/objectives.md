# Цели обучения - Сценарий: Защита операционных систем

## Общие цели

Сценарий "Защита операционных систем" разработан для практического обучения навыкам обнаружения, анализа и устранения типичных уязвимостей операционных систем. Инфраструктура включает три виртуальные машины с различными операционными системами (Ubuntu Server, Windows Server 2016, Windows 10 Pro), настроенными с реалистичными уязвимостями ОС.

## Цели обучения по уровням

### Начальный уровень

#### 1. Обнаружение уязвимостей аутентификации
**Цель**: Научиться находить типичные уязвимости аутентификации в операционных системах

**Навыки**:
- Обнаружение слабых паролей
- Проверка политики паролей
- Выявление небезопасных методов аутентификации
- Анализ конфигураций SSH/RDP

**Практические задачи**:
- Найдите слабые пароли на всех машинах
- Обнаружьте разрешенный root login на Ubuntu
- Проверьте политику паролей на Windows
- Найдите учетные записи с небезопасными настройками

#### 2. Проверка управления обновлениями
**Цель**: Научиться проверять статус обновлений безопасности и выявлять устаревшие системы

**Навыки**:
- Проверка статуса автоматических обновлений
- Определение устаревших версий ОС и компонентов
- Выявление отсутствующих патчей безопасности
- Анализ политик обновлений

**Практические задачи**:
- Проверьте статус автоматических обновлений на всех машинах
- Найдите устаревшие версии компонентов ОС
- Определите отсутствующие патчи безопасности
- Проверьте настройки Windows Update и apt

### Средний уровень

#### 3. Аудит прав доступа
**Цель**: Научиться анализировать права доступа к файлам и обнаруживать неправильные настройки

**Навыки**:
- Анализ прав доступа к файлам (Linux и Windows)
- Обнаружение учетных данных в открытом виде
- Проверка прав доступа к чувствительным директориям
- Выявление избыточных привилегий

**Практические задачи**:
- Найдите файлы с неправильными правами доступа (777, FullControl для Everyone)
- Обнаружьте учетные данные в открытом виде
- Проверьте права доступа к чувствительным директориям
- Выявите пользователей с избыточными привилегиями

#### 4. Обнаружение встроенных уязвимостей ОС
**Цель**: Научиться выявлять уязвимые протоколы и небезопасные настройки ОС

**Навыки**:
- Проверка включенных уязвимых протоколов (SMBv1, NTLMv1)
- Анализ статуса брандмауэра
- Проверка статуса антивируса/защиты
- Выявление небезопасных сетевых настроек

**Практические задачи**:
- Проверьте включен ли SMBv1 на всех машинах
- Проверьте статус брандмауэра Windows
- Проверьте статус антивируса/защиты
- Найдите небезопасные сетевые протоколы

### Продвинутый уровень

#### 5. Исправление уязвимостей
**Цель**: Научиться эффективно устранять обнаруженные уязвимости и усиливать безопасность

**Навыки**:
- Применение исправлений для уязвимостей
- Настройка правильной политики паролей
- Включение и настройка автоматических обновлений
- Исправление прав доступа к файлам
- Отключение уязвимых протоколов и сервисов

**Практические задачи**:
- Исправьте все обнаруженные уязвимости
- Настройте правильную политику паролей на всех системах
- Включите автоматические обновления
- Исправьте права доступа к файлам
- Отключите уязвимые протоколы и сервисы (SMBv1, NTLMv1 и др.)

#### 6. Комплексный аудит безопасности ОС
**Цель**: Научиться проводить полный аудит безопасности операционных систем

**Навыки**:
- Комплексный анализ безопасности ОС
- Документирование найденных уязвимостей
- Приоритизация уязвимостей по критичности
- Составление плана устранения уязвимостей

**Практические задачи**:
- Проведите полный аудит безопасности всех машин
- Составьте отчет о найденных уязвимостях
- Приоритизируйте уязвимости по критичности
- Разработайте план устранения уязвимостей

## Результаты обучения

После завершения всех упражнений студенты должны уметь:

### Технические навыки

1. **Обнаружение уязвимостей ОС**
   - Использовать инструменты для аудита безопасности
   - Анализировать конфигурации ОС
   - Выявлять типичные ошибки безопасности
   - Находить уязвимости в Linux и Windows системах

2. **Анализ конфигураций**
   - Анализировать настройки аутентификации
   - Проверять политики безопасности
   - Оценивать права доступа
   - Выявлять небезопасные протоколы

3. **Устранение уязвимостей**
   - Применять исправления безопасности
   - Настраивать безопасные конфигурации
   - Усиливать безопасность систем
   - Валидировать исправления

### Специфичные навыки

1. **Работа с Linux (Ubuntu)**
   - Понимание системы прав доступа
   - Настройка SSH безопасности
   - Управление обновлениями через apt
   - Работа с файловой системой и правами

2. **Работа с Windows**
   - Понимание политик безопасности (GPO)
   - Настройка Windows Update
   - Управление правами доступа
   - Работа с брандмауэром и защитой

3. **Сравнительный анализ**
   - Сравнение подходов к безопасности в разных ОС
   - Понимание общих принципов безопасности
   - Адаптация навыков к различным системам

### Мягкие навыки

1. **Аналитическое мышление**
   - Анализировать сложные конфигурации
   - Выявлять взаимосвязи между настройками
   - Принимать решения на основе анализа

2. **Документирование**
   - Ведение записей о процессе аудита
   - Составление отчетов об уязвимостях
   - Коммуникация результатов

3. **Системный подход**
   - Проведение комплексного аудита
   - Приоритизация задач
   - Планирование устранения уязвимостей

## Критерии оценки

### Отлично (5)
- Обнаружил все уязвимости на всех машинах
- Провел полный анализ конфигураций
- Успешно устранил все уязвимости
- Составил подробный отчет с рекомендациями
- Продемонстрировал глубокое понимание безопасности ОС

### Хорошо (4)
- Обнаружил большинство уязвимостей
- Провел анализ основных конфигураций
- Устранил большинство уязвимостей
- Составил хороший отчет
- Продемонстрировал базовое понимание безопасности ОС

### Удовлетворительно (3)
- Обнаружил основные уязвимости
- Провел базовый анализ
- Устранил некоторые уязвимости
- Составил базовый отчет

### Неудовлетворительно (2)
- Не обнаружил уязвимости или обнаружил минимальное количество
- Не провел анализ конфигураций
- Не устранил уязвимости
- Не составил отчет или отчет не соответствует требованиям

## Временные рамки

### Начальный уровень
- Обнаружение уязвимостей аутентификации: 1-2 часа
- Проверка управления обновлениями: 1 час
- **Итого**: 2-3 часа

### Средний уровень
- Аудит прав доступа: 2-3 часа
- Обнаружение встроенных уязвимостей: 1-2 часа
- **Итого**: 3-5 часов

### Продвинутый уровень
- Исправление уязвимостей: 2-3 часа
- Комплексный аудит: 2-3 часа
- **Итого**: 4-6 часов

### Полный сценарий
- **Общее время**: 9-14 часов (в зависимости от уровня подготовки)

## Рекомендации по проведению

### Подготовка
1. Убедитесь, что все виртуальные машины развернуты и настроены
2. Проверьте доступность всех систем
3. Подготовьте инструкции для студентов
4. Настройте систему мониторинга (опционально)

### Проведение
1. Предоставьте четкие инструкции по началу работы
2. Обеспечьте техническую поддержку
3. Мониторьте прогресс студентов
4. Предоставляйте подсказки при необходимости
5. Обратите внимание на различия между Linux и Windows

### Завершение
1. Проведите разбор результатов
2. Обсудите ключевые моменты безопасности ОС
3. Ответьте на вопросы
4. Соберите обратную связь
5. Обсудите лучшие практики безопасности
